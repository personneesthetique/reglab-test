# Reglab Test

## Запуск приложения

```bash
npm start
```

Бэкенд имитируется при помощи localStorage и sessionStorage

Для реализации бэкенда использовался:

1. Broadcast Channel - для отправки и получения сообщения в реальном времени (имитация WebSocket)
2. Polling - для получения и обновления статуса/ов (имитация отправки сообщения при открытии/закрытии WebSocket-соединения)

Пользователи системы:

- логин: user1, пароль: user1
- логин: user2, пароль: user2
- логин: user3, пароль: user3
